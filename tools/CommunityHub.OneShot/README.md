# CommunityHub.OneShot

On-demand CLI runner for hub jobs. Drives the **same** service classes the
Azure Function App drives, so what you run here matches what runs in
production. Use it to verify a pipeline locally before deploying the
Function App, or when chasing a bug that needs an extra ad-hoc run.

## Commands

| Command          | What it does                                                                 |
|------------------|------------------------------------------------------------------------------|
| `pull-sponsors`  | Runs `SponsorOrderPullService.RunAsync()` once: fetches completed WooCommerce orders, enriches line items with categories, classifies, expands into sponsor tasks, upserts ParticipantTask rows. Idempotent. |

## DEV email safety

When run with `DOTNET_ENVIRONMENT=Development`, `appsettings.Development.json`
sets `Email:RedirectAllTo = mok@expertslive.dk`. Any outbound mail
generated by a OneShot run is rerouted to that inbox; the original
recipient is preserved in the subject as `[TEST -> original@x] Original`.

Never deploy `appsettings.Development.json` to production.

## Required configuration

Populated via env vars / user-secrets / appsettings:

| Key                              | Required for          | Notes                                                  |
|----------------------------------|-----------------------|--------------------------------------------------------|
| `Sql__ConnectionStringTemplate`  | all commands          | e.g. `Server=tcp:eldk27hub-sql-...;Database=eldk27hub-db;Encrypt=True;` |
| `Sql__AdminPassword`             | all commands          | KV: `ELDKHUB-SQL-ADMIN-PASSWORD`                      |
| `WooCommerce__ConsumerKey`       | `pull-sponsors`       | KV: `woocommerce-consumer-key` (read-only key)        |
| `WooCommerce__ConsumerSecret`    | `pull-sponsors`       | KV: `woocommerce-consumer-secret`                     |
| `WooCommerce__BaseUrl`           | `pull-sponsors`       | Defaults to `https://expertslive.dk`                  |
| `CompanyManager__Username`       | `pull-sponsors` (contact sync) | KV: `company-manager-wp-user`                 |
| `CompanyManager__Password`       | `pull-sponsors` (contact sync) | KV: `company-manager-wp-app-password`         |
| `CompanyManager__BaseUrl`        | `pull-sponsors` (contact sync) | Defaults to `https://expertslive.dk/wp-json/company-manager/v1` |
| `Email__SmtpUsername`            | future mail commands  | KV: `brevo-smtp-username`                             |
| `Email__SmtpKey`                 | future mail commands  | KV: `brevo-smtp-key`                                  |

## Run it

```powershell
# Set env once per shell
$env:DOTNET_ENVIRONMENT          = "Development"
$env:Sql__ConnectionStringTemplate = "Server=tcp:eldk27hub-sql-devz237e.database.windows.net,1433;Database=eldk27hub-db;Encrypt=True;TrustServerCertificate=False;Connection Timeout=30;"
$env:Sql__AdminPassword          = "<from KV>"
$env:WooCommerce__ConsumerKey    = "<from KV>"
$env:WooCommerce__ConsumerSecret = "<from KV>"

dotnet run --project tools/CommunityHub.OneShot -- pull-sponsors
```

The run logs orders fetched + tasks created. Re-running is safe: the
SourceKey idempotence in `SponsorOrderPullService` skips tasks that
already exist.
